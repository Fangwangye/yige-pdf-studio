# 翻译服务商接入

## 现有：OpenAI-compatible

通过 `pdf2zh` 的 `openailiked` 服务实现。前端在「接入设置」填：

- **Base URL**：填到 `/v1` 这层，如 `https://api.deepseek.com/v1`。
- **Model**：如 `deepseek-v4-pro`。
- **API Key**：`sk-...`。

凭证经 `POST /api/translate` 传入，封装进 `Pdf2zhConfig`，由 `pdf2zh_engine.translate_with_pdf2zh` 拼成 `pdf2zh` 命令行调用。

## Argos 离线翻译（开源 · 无需 Key）

`provider=argos`：走 `pdf2zh` 内置的 `ArgosTranslator`（神经机器翻译，offline），**仍由 babeldoc 重建版式**。

- 依赖：`argostranslate`（见 requirements，可选）。首次运行联网下载 en→zh 语言包，之后离线。
- 实现：仅走 babeldoc 路径（engine 内 `use_babeldoc` 对 argos 强制为真）。入口 `app/pdf2zh_babeldoc_cli.py` 在调 `yadt_main` 前用 `_patch_argos_translator()` 修复 pdf2zh 1.9.11 中 `ArgosTranslator.translate` 的导入笔误，并改用带缓存的 `do_translate`。
- 不需要 api_key / base_url / model（engine 对 argos 跳过这三项校验）。
- **限制**：Argos 是 NMT，不吃提示词，**知识库（术语/风格/禁译）对它不生效**，质检报告知识库项显示「未应用」。要用知识库请用 OpenAI-compatible provider。
- 不触发 legacy 自动回退（legacy 走真实 pdf2zh 二进制，无法打补丁）。

## Mock 模式

`provider=mock` 时走 `copy_pdf_without_translation`：只复制原 PDF 到输出，用于验证上传/预览/下载与版式，不调用模型、不耗 token。

## 新增一个服务商

1. 确认目标是 OpenAI-compatible：多数情况直接换 Base URL/Model/Key 即可，无需改代码。
2. 若协议不同（非 OpenAI-compatible）：
   - 在 `pdf2zh_engine.py` 增加对应的引擎分支/服务名映射。
   - 在 `static/index.html` 接入设置的 `#provider` 增加选项。
   - 在 `app/main.py` 的参数与 `Pdf2zhConfig` 透传新字段。
3. 在本文件补一行配置示例。

## 提示词与知识库

最终 prompt = 系统翻译指令 + 自动文档上下文 + 用户知识库（优先级更高），由 `pdf2zh_engine.default_translation_prompt` 拼装并写临时 prompt 文件，经 `--prompt` 传给 `pdf2zh`。新增服务商无需重写这套逻辑。
