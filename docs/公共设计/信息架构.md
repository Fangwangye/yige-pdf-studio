# 信息架构

## 布局

左侧固定导航 + 右侧工作区（`static/index.html` 的 `.layout`）。所有控件常驻 DOM，仅通过 `.view.is-active` 切换可见性——因此「开始翻译」在任意视图都能读到设置与知识库的值。

```
┌─────────────┬───────────────────────────────────┐
│  Sidebar    │  Topbar（标题/副标题 + 下载/打开） │
│  - 品牌     ├───────────────────────────────────┤
│  - 导航×5   │  view-scroll                      │
│  - 连接状态 │    └ 当前激活的 view               │
└─────────────┴───────────────────────────────────┘
```

## 五个视图

| 视图 | data-panel | 职责 | 关键元素 ID |
| --- | --- | --- | --- |
| 翻译工作台 | `workspace` | 主流程：选文件→设语言→选知识库→翻译→对照预览 | `pdfFile` `sourceLanguage` `targetLanguage` `knowledgeProfile` `translateButton` `sourcePreview` `translatedPreview` `statusLine` |
| 知识库 | `knowledge` | 维护术语/风格/禁译规则，增删改导入导出 | `knowledgeProfileName` `knowledgeBase` `save/new/export/import/deleteKnowledgeProfile` |
| 任务与质检 | `jobs` | 任务历史 + 质量报告 | `jobHistory` `refreshJobs` `qualityReport` `qualityMeta` |
| 接入设置 | `settings` | 服务商/Key/版式/质量 | `provider` `baseUrl` `model` `apiKey` `layoutEngine` `qualityMode` `preserveToc` `protectedPages` |
| 帮助 | `help` | 快速上手 + 路线图 | — |

## 导航交互

- 切换由 `app.js` 的 `switchView(name)` 驱动：同步高亮 nav、显示对应 view、更新 topbar 标题/副标题（`viewMeta`）。
- 正文中的 `[data-view]`（非 nav 的内链）也能跳视图，例如知识库提示里指向「帮助」。

## 主流程（happy path）

```
选 PDF（左栏即时预览） → 选语言/知识库 → 开始翻译
   → POST /api/translate 拿 job_id
   → 轮询 GET /api/jobs/{id} 直到 succeeded
   → 右栏加载 /api/preview/{id}，刷新质检报告与历史
```

## 设计原则

- **主流程优先**：工作台只放最常用的 4 个控件 + 翻译按钮，重配置收进「接入设置」。
- **不丢状态**：视图切换不卸载控件，翻译随时可触发。
- **结果可见**：状态栏 + 质检报告让「翻得怎么样」可观测。
