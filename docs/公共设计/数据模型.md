# 数据模型

## Job（任务状态，服务端持久化）

存于 `storage/jobs/{job_id}.json`，由 `app/main.py` 读写。关键字段：

| 字段 | 说明 |
| --- | --- |
| `job_id` | uuid hex |
| `state` | `queued` / `running` / `succeeded` / `failed` |
| `source_filename` | 原始文件名 |
| `created_at` / `updated_at` / `started_at` / `finished_at` | ISO 时间 |
| `settings` | 本次翻译配置快照（语言、provider、model、引擎、质量、保留页、知识库是否应用） |
| `stats` | 翻译结果统计（见下） |
| `error` | 失败原因 |
| `log_tail` | 引擎日志尾部（前端轮询展示） |
| `*_url` | `source/preview/download/status` 等便捷链接 |

### stats

`pages`、`engine`、`preserved_pages`、`fallback_pages`、`quality_warnings[{page,...}]`、`page_bridges`、`context_terms`、`knowledge_base_applied`。

## QualityReport（质检报告，派生）

`GET /api/jobs/{id}/report` 由 `_build_quality_report` 实时计算，不单独存储。包含 `checks[]`（名称/状态 pass|warn|idle|pending/详情）、`warnings`、`fallback_pages`、`preserved_pages`。

## KnowledgeProfile（知识库）

### 当前（纯文本，过渡态）
前端 `localStorage["pdfTranslateKnowledgeProfiles"]`：`{ [name]: "纯文本规则" }`；翻译时整段拼进 prompt。

### 目标（一期，结构化）
迁到服务端 `storage/knowledge/{name}.json`：

```jsonc
{
  "name": "学术论文",
  "version": 2,
  "glossary": [
    { "src": "large language model", "dst": "大语言模型", "case_sensitive": false, "note": "" }
  ],
  "style_rules": ["使用正式流畅的学术中文"],
  "do_not_translate": ["模型名", "公式", "引用"],
  "updated_at": "2026-06-30T00:00:00Z"
}
```

- **向后兼容**：导入旧的 `{name, content}` 纯文本时，整段塞进 `style_rules`。
- **命中注入**：翻译前扫描文档文本，仅注入 `glossary` 中命中的条目 + 全部 `style_rules` / `do_not_translate`。

## DocumentContext（文档级上下文，自动抽取）

`pdf2zh_engine.build_document_context` 产出：`title`、`abstract`、`terms[]`（高频术语）、`page_bridges`（跨页断句）。优先级**低于**用户知识库，一并注入 prompt。
