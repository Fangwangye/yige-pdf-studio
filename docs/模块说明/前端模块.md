# 前端模块（static/）

原生 HTML/CSS/JS，无构建步骤。三件套：`index.html` 结构、`styles.css` 样式、`app.js` 逻辑。

## index.html
- `.layout` = 左侧 `.sidebar`（品牌 + 5 个 `.nav-item[data-view]` + 连接状态）+ 右侧 `.workspace`（`.topbar` + `.view-scroll`）。
- 5 个 `.view[data-panel]`：workspace / knowledge / jobs / settings / help，仅一个带 `.is-active`。
- **所有表单控件常驻 DOM**，切视图只改可见性 → 翻译按钮在任意视图都能读到设置与知识库的值。
- 改样式/脚本后 bump `?v=` 破缓存。

## styles.css
以 `:root` CSS 变量为单一事实源，详见 [公共设计/UI设计规范.md](../公共设计/UI设计规范.md)。

## app.js
职责分块：

| 区块 | 关键函数 |
| --- | --- |
| 元素引用 | 顶部一批 `querySelector`（ID 见 [信息架构.md](../公共设计/信息架构.md)） |
| 翻译提交 | `translateButton` 监听 → 组 `FormData` → `POST /api/translate` |
| 任务轮询 | `pollJob`：每 1.6s 查状态，按 queued/running/failed/succeeded 更新 UI |
| 预览加载 | `loadJobPreview`：设 `sourcePreview`/`translatedPreview` 的 `src`（带 cacheBust） |
| 任务历史 | `refreshJobHistory` / `renderJobHistory` |
| 质检报告 | `renderQualityReport` |
| 知识库 | `localStorage` 读写 + `renderKnowledgeProfiles`（增删改导入导出） |
| 视图导航 | `switchView` + `viewMeta`（更新 nav 高亮与 topbar 标题） |
| 连接指示 | `refreshConnState`（Mock / 未配 Key / 已连接） |

## 注意
- 知识库当前存浏览器 `localStorage["pdfTranslateKnowledgeProfiles"]`，换设备不同步。一期迁服务端后，知识库读写改为调 `/api/knowledge`（见 [知识库设计.md](./知识库设计.md)）。
- 新增视图：加 `.nav-item[data-view=x]` + `.view[data-panel=x]`，并在 `viewMeta` 补标题/副标题。
