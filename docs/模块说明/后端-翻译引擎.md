# 后端 · 翻译引擎（app/pdf2zh_engine.py）

封装对 `pdf2zh` / BabelDOC 的调用，以及提示词拼装、文档上下文抽取、质检与异常页回退。

## 配置
`Pdf2zhConfig`（dataclass）：语言、API 凭证、`thread_count`、`use_babeldoc`、`preserve_toc`、`protected_pages`、`prompt`、`knowledge_base`。

## 主流程 `translate_with_pdf2zh` / `_translate_with_pdf2zh_sync`
1. `build_document_context` 抽取文档级上下文（标题/摘要/高频术语/跨页断句）。
2. `default_translation_prompt(...)` 拼系统指令 + 文档上下文 + 用户知识库 → `write_prompt_file` 落临时文件，经 `--prompt` 传给 `pdf2zh`。
3. 运行 `pdf2zh`（`_run_command`，日志回调）。
4. **质检 + 回退**（BabelDOC 路径）：
   - `scan_pdf_quality` 扫描占位符/乱码/替换字符 → `initial_quality_warnings`。
   - 若有问题页：`run_legacy_fallback` 用 legacy 引擎重跑这些页 → `replace_pages` 替换回最终 PDF。
   - 回退后 `scan_pdf_quality` 再扫一次得最终 `quality_warnings`。
5. 返回 `stats`：`pages`、`engine`、`preserved_pages`、`fallback_pages`、`fallback_error`、`quality_warnings`、`page_bridges`、`context_terms`、`knowledge_base_applied`。

## 提示词 `default_translation_prompt`
- 系统指令固定若干规则（保占位符/公式/引用、术语一致、跨页上下文只作参考等）。
- `context_block`：文档级上下文。
- `knowledge_block`：用户知识库，**优先级高于**文档上下文，但不覆盖保护规则。
- `normalize_knowledge_base` 对知识库做长度截断（默认 6000）。

## 文档上下文 `build_document_context`
`_extract_title` / `_extract_abstract` / `_extract_recurring_terms`（高频且有意义的术语）/ `_extract_page_bridges`（识别页尾→下页页首的续句，减少跨页断裂）。

## 质检 `scan_pdf_quality`
扫描输出 PDF 的疑似异常（未解析占位符、乱码、`�` 替换字符），按页返回 `{page, ...}`。

## 回退 `run_legacy_fallback` / `replace_pages`
- legacy 引擎对问题页单独重跑，取 `*mono*.pdf` 产物。
- `replace_pages` 把指定页替换进最终 PDF（临时文件 + 替换）。

## 维护注意
- 改动 stats 字段需同步：`main.py` 的 `_job_summary` / `_build_quality_report`、前端 `pollJob` 的展示。
- 二期 MCP 检索的注入点在 `default_translation_prompt` 的 `knowledge_block`：把静态文本换成检索结果即可，不动其余流程。
