# 产品概述

## 一句话

把英文（及其他语种）PDF 翻译成中文，**保留原始版式**（图片、表格、公式、页面布局），并用**知识库**保证术语和风格一致。

## 目标用户

- 需要读外文论文 / 技术文档 / 合同的研究者、工程师、译者。
- 对术语一致性、版式还原有要求，单纯机翻无法满足。
- 希望本地运行、数据不外泄（仅翻译文本经过模型 API）。

## 核心价值

1. **版式重建**：基于 PDFMathTranslate / pdf2zh / BabelDOC，先解析版面再翻译重排，而非粗暴覆盖文本。
2. **左右对照**：原文 PDF 与译文 PDF 同屏预览，便于校对。
3. **知识库一致性**：术语对照表 + 风格规则 + 禁译表，注入翻译提示词，跨页、跨文档保持一致。
4. **文档级上下文**：自动抽取标题、摘要、高频术语与跨页断句上下文，减少长文逻辑断裂。
5. **质量可见**：占位符 / 乱码扫描、异常页自动回退、跨页连续性统计，结果以质检报告呈现。

## 功能清单

| 模块 | 能力 |
| --- | --- |
| 翻译工作台 | 上传 PDF、设语言、选知识库、开始翻译、左右对照预览、下载 |
| 知识库 | 多套配置、术语/风格/禁译规则、保存/导入/导出（当前存浏览器本地） |
| 接入设置 | OpenAI-compatible 服务商、Base URL/模型/Key、版式引擎、质量模式、目录页保护 |
| 任务与质检 | 后台任务历史、状态轮询、质量报告（占位符扫描/回退/跨页/知识库命中） |

## 运行形态

- 后端：FastAPI（单机本地服务，默认 `127.0.0.1:8010`）。
- 前端：原生 HTML/CSS/JS，单页 + 左侧导航。
- 翻译：调用本地 `pdf2zh` / BabelDOC + 远端 OpenAI-compatible 模型。

## 当前限制

- 知识库是**纯文本规则**、存浏览器 `localStorage`，尚未结构化、未做命中检索、不跨设备同步（一期目标）。
- `pdf2zh` 首次运行需下载模型/字体，较慢。
- 扫描版 PDF 需 OCR 路径，暂未内建。
- 默认质量优先（线程数 1），速度偏慢。
- PDF 非结构化格式，无法对所有文件承诺 100% 版式一致。
