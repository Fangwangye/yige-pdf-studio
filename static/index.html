标准使用流程

第一次用按这五步走，之后重复第 3–5 步即可。

  1. 配置服务商（接入设置）：填 Base URL / 模型 / API Key；没有 Key 想先看效果就选 Argos 离线或 Mock。
  2. 准备知识库（知识库）：选或新建一套术语/风格/禁译规则，保存；决定翻译的专业度与一致性。
  3. 选文档类型 + 分段（接入设置 → 文档结构）：选类型自动套知识库，勾选要保留原文的段落（目录/参考文献等）。
  4. 上传并翻译（翻译工作台）：选 PDF → 设语言 → 开始翻译；长文档把并发线程调到 8–16 更快。
  5. 对照与质检：用「对照模式」左右同步核对；到「任务与质检」看占位符/乱码扫描与命中术语数。

翻译时发生了什么

了解流程有助于判断在哪一步调优。

上传 PDF→ 版面解析（BabelDOC）→ 抽取文档上下文→ 注入知识库/术语→ 分块翻译并重排→ 质检 / 异常页回退→ 生成译文 PDF

分场景建议

  • 学术论文：文档类型选「学术论文」，勾选保留「参考文献/附录」；术语一致性要求高时用 OpenAI-compatible + 知识库（Argos 不吃术语）。
  • 技术文档：把 API/SDK/命令/配置项放进「禁译」；代码块勾选保留原文。
  • 合同：条款编号/金额/日期加入禁译，风格选正式稳健，逐条核对不要弱化义务。
  • 只想快速读懂：可接受版式一般，优先把并发调大、质量模式选「速度优先」。

遇到问题怎么排查

  • 右侧没出译文 → 到「任务与质检」看任务状态；failed 会显示引擎错误摘要。
  • 术语没按知识库翻 → 确认翻译前已选对应知识库；且服务商不是 Argos（NMT 不吃提示词）。
  • 太慢 → 接入设置把「并发线程」调到 8–16（注意服务商限流）；首次运行还要下载模型。
  • 乱码/占位符 → 质检报告定位到具体页，必要时切 legacy 引擎重试。
  • 跨页/跨栏断裂 → 保版式的固有限制；可开「改善页衔接（实验）」缓解段内碎片。
